# 达人建联功能 — 开发计划

## 1. 功能总览

在现有 TikTok 运营看板中新增「达人建联」页面，管理达人合作关系，追踪样品寄送物流，自动提醒已送达样品。

---

## 2. 功能模块

### 2.1 达人管理
| 功能 | 说明 |
|------|------|
| 添加/编辑/删除达人 | 姓名、平台账号、联系方式、粉丝量、品类标签、备注 |
| 合作状态 Pipeline | 待联系 → 已联系 → 洽谈中 → 已确认 → 样品已寄 → 内容已收到 → 已发布 → 已结算 |
| 达人列表 | 表格展示，支持按状态、标签筛选，搜索 |
| 达人详情 | 点击展开：沟通记录、样品记录、内容链接、历史合作汇总 |

### 2.2 样品物流追踪
| 功能 | 说明 |
|------|------|
| 录入追踪号 | 选择达人 → 填写快递单号 → 选择快递公司 |
| 自动查询物流状态 | 调用第三方物流 API 查询：待揽收 / 运输中 / 已签收 / 异常 |
| 物流状态列表 | 所有包裹当前状态一览，按状态排序 |
| 一键跳转官网 | 提供快递公司官网追踪链接快速跳转 |

### 2.3 到货提醒
| 功能 | 说明 |
|------|------|
| 已送达提醒区 | 页面顶部醒目区域展示「已签收」包裹列表，显示达人 + 样品 + 签收时间 |
| 已处理标记 | 达人确认收到后可标记「已处理」，归档到历史 |
| 未读角标 | 侧边栏菜单红点提示未处理到货数量 |

### 2.4 沟通记录
| 功能 | 说明 |
|------|------|
| 添加沟通日志 | 日期、方式（私信/邮件/微信）、内容摘要 |
| 时间线展示 | 按时间倒序展示每条沟通记录 |

### 2.5 数据统计
| 功能 | 说明 |
|------|------|
| Pipeline 概览 | 各阶段达人数量卡片 |
| 样品寄送统计 | 寄出/在途/已签收数量 |
| 合作转化率 | 联系→确认→发布 的漏斗转化 |

## 3. 物流追踪实现方案（待确认）

前端的局限：浏览器直接请求快递公司官网会被 CORS 阻止。

3 种方案供选择：

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **A: 手动更新** | 用户自己查物流后手动标记状态 | 零成本、零依赖 | 需要人工操作 |
| **B: Vercel Serverless 代理** | 在 Vercel 写一个 `/api/track` 函数，后端代理调用快递鸟/快递100 API | 自动化，体验好 | 需要注册物流API账号（快递鸟免费额度够用） |
| **C: 仅提供快捷链接** | 不查询 API，点击追踪号直接跳转快递100查询页 | 简单 | 无法自动提醒 |

**建议方案 B**，快递鸟免费版日调用 500 次，对小规模运营足够。

### 4.3 数据模型

```typescript
// 达人
interface Influencer {
  id: string;
  name: string;
  platform: string;           // TikTok / Instagram / YouTube
  handle: string;             // @username
  followers: number;
  category: string[];         // 品类标签
  contactInfo: string;        // 联系方式
  status: PipelineStage;      // 当前合作阶段
  notes: string;
  createdAt: string;
  updatedAt: string;
}

// 合作阶段
type PipelineStage = 'pending' | 'contacted' | 'negotiating' | 'confirmed' | 'sample_sent' | 'content_received' | 'published' | 'settled';

// 物流包裹
interface Shipment {
  id: string;
  influencerId: string;
  influencerName: string;
  productName: string;        // 寄送样品名称
  trackingNumber: string;
  carrier: string;            // 快递公司
  status: TrackingStatus;     // 物流状态
  statusUpdatedAt: string;
  deliveredAt: string | null;
  acknowledged: boolean;      // 达人确认收到
  createdAt: string;
}

// 物流状态
type TrackingStatus = 'pending' | 'in_transit' | 'delivered' | 'exception';

// 沟通记录
interface CommLog {
  id: string;
  influencerId: string;
  date: string;
  method: string;             // 私信/邮件/微信/电话
  content: string;
}
```
